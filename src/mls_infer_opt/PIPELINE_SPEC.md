本文件描述调优流水线**应该长成的样子**——按"这条流水线本质是什么"从头推导，不从现有代码反推。

## 0. 流水线的本质

**对一段候选程序做有界搜索。**

在一个时间墙内，反复「求梯度（定方向）→ 合成候选（训练一步）→ 验证度量」，最终发布最好的候选，
并保证无论中途发生什么都有一个可交付物。沿用训练类比：这是一个**带昂贵 / 易错 oracle 的小型训练循环**。

它不是简单的数据 ETL 管道，而是优化循环。这个定性决定了三条底层事实，所有设计都从它们出发：

1. **失败是常态，不是异常**——候选跑不对、跑得慢是循环赖以运转的信号，而非故障。
2. **预算是一等公民**——时间 / 调用次数 / 评测次数有限，每个阶段都在花，总控负责记账与判停。
3. **可复现与可发布终态是硬约束**——任意中断点都要能交付一个已验证的 best。

**核心切分：总控 + 行为阶段。** 沿用训练循环：`grad / train / eval` 是**行为阶段**（每轮被调用的动作，
纯变换 输入→输出，不持有控制权）；`trainer` 是**总控**（持有状态、预算、控制流、终止与发布）。
区分这两类是整份 spec 的骨架——**行为阶段只产信号，控制决定全在总控**。

| 训练类比 | 模块 | 类别 | 职责 |
|---|---|---|---|
| trainer | loop | **总控** | 装配 / 驱动循环 / keep-best / 停止准则 / 发布 |
| grad | analyze | 行为阶段 | 看反馈算下一步方向（`Move` \| `NoMove`），**不判停** |
| train | generate | 行为阶段 | 按方向产候选（含不依赖 LLM 的初始 step = baseline） |
| eval | evaluate | 行为阶段 | 正确性硬门 + 性能度量 |

## 1. 总控与行为阶段的完整性清单

每项按同一组完整性轴审视：**输入/输出契约、失败语义、确定性、幂等/缓存、预算消耗、必须留痕**。

### 总控（trainer）

唯一持有主状态、预算、控制流的组件。子职责：

- **装配（init）**：确定性 run 身份；environment **一次探测、之后只读**；**LLM 模式一次性裁定**
  （已配置 → 全程要求可用；未配置 → 全程显式 rule-based，运行中不再中途降级）；前置校验
  （weights / model_config 在不在、合法不合法）。
- **驱动循环**：预算记账；每轮按 求梯度 → 合成 → 验证 序列化调用；安全上限防空转。
- **keep-best（checkpoint）**：单一选优逻辑、严格更优、过门只是入场券；提升即**增量发布**（抗 kill）；
  best 一旦有不退化。**纯确定性比较，无外部依赖**——出异常即 C4 bug。
- **停止准则（stopping criterion）**：硬停（预算 / max_rounds / 时间墙 = C3）+ 收敛早停（stale patience）
  + 收行为阶段返回的 `NoMove`；**由总控裁决终止、写 stop_reason**。
- **发布（export）**：唯一发布出口、以正确性门为硬条件；**截止时也保证可发布终态**；
  最外层 finally 保证 baseline 兜底。

总控不产候选、不算方向、不跑评测；行为阶段不判停、不发布、不改状态。

### 行为阶段 ①：求梯度 / 定方向（grad）

- **契约**：状态只读视图 → `Move(policy)` | `NoMove(rationale)`。
  **永不判停**——gradient 只给方向，停止是总控的循环准则。
- **配齐**：
  - 方向 = 从 best 出发的 axes / knobs delta。
  - `NoMove` = gradient ≈ 0（无产出方向），带 rationale 交总控，作为一个**终止输入值**（不是终止动作）。
  - 只读、无发布权、不改状态。
- **失败语义**：建视图崩 = C4；**LLM 调用失败 = C2 报**；LLM 给了合法但平庸的答案 = 回规则阶梯（C1 邻域）。

### 行为阶段 ②：合成 / 训练一步（train）

- **契约**：方向 + 父工件 → 候选工件（落盘）+ 元数据，或"这轮没产出"。
- **配齐**：
  - **稳定 id**（审计 / 落盘目录 / report 引用；**不为去重缓存**——见 §2 B）。
  - **昂贵评测前的契约早筛**（自包含 / API），且"什么算合法候选"这份定义**与验证阶段共享一份**。
  - **保底产出途径**（baseline = 不依赖 LLM 的初始 step），保证"永远有可发布物"。
- **失败语义**：**LLM 调用失败 = C2 报**；候选自检 / 契约不过 = C1 正常的"这轮没收益"。

### 行为阶段 ③：验证 / 度量（eval）

- **契约**：候选 → 正确性硬裁决 + 质量度量（仅过门才有）。
- **配齐**：
  - **崩溃隔离**（坏候选不能拖垮总控）——**唯一**合理的 never-throw 区（对 C1）。
  - **同一候选评测幂等**（gate 已填则不重测）——一个判空，非内容缓存。
  - **确定性 + reference 复用**（固定 seed；reference logits 算一次、跨候选复用 = oracle 缓存，按 mode，与候选身份无关）。
  - **结构化失败**（stage / case / err / shape）供 求梯度 与 合成 消费。
  - 正确性与性能**物理分离**。
- **失败语义**：候选崩 = C1 结构化裁决（信号）；
  **评测器进程根本起不来 = C2**，要与"候选 runtime 失败"区别对待，不可糊成一个 failed gate。

## 2. 横切点

只有出现在 ≥2 项"配齐"里的，才算真横切。

| 横切点 | 贯穿哪些 | 它统一的东西 |
|---|---|---|
| **A. 失败分类法** | 全部 | 见 §3，总纲 |
| **B. 候选身份** | 合成铸造 → keep-best 比较 → 报告引用 → 落盘寻址 | 一份 id 语义（审计/引用/确定性；**不做缓存去重**） |
| **C. 预算 / 截止 + 可发布终态保证** | 行为阶段消耗 → 总控 enforce → 发布 / finally 兜底 | 一份预算账 + 终态不变量 |
| **D. 工件存储与寻址** | 合成写 → 验证读 → 发布 → 总控归档 | 一份命名 / IO 契约 |
| **E. 可观测留痕** | 每项记决策 → 报告 / 调试 / 抗 kill 重放 | 一个 emit + 一份事件契约 |
| **F. 确定性 + 血缘** | 缓存 / 复现（hash / seed / 稳定序）+ 谱系（parent / kind / round） | 一份可复现与可追溯约定 |

## 3. 贯穿原则：失败分类法与 never-throw 边界

整条流水线只有**四类失败**，各走各的策略。把它们混成一类（尤其把 C2 糊进 C1 的"never-throw → 降级"）
是当前最大的设计债。

| 类别 | 例子 | 策略 |
|---|---|---|
| **C1 候选域失败** | 候选 logits 不对、候选自身崩、跑得慢 | **是信号，不是错误**。结构化捕获、喂回循环。绝不抛。 |
| **C2 基建失败** | **LLM 调用失败**、评测器起不来、weights 缺失、写盘失败 | **报错 / 显式中止**。绝不伪装成 C1 被吞掉。 |
| **C3 截止** | 时间墙、max_rounds | 计划内退出，**保证干净可发布终态**。 |
| **C4 总控 bug** | 编排 / keep-best 逻辑里的真异常 | 开发期炸响；最外层 finally 仍保证盘上有 baseline。 |

**原则：never-throw 只许包两处——C1（候选域，集中在 eval 崩溃隔离）和最外层发布兜底（C4 的 finally）。
C2 必须穿透。**

推论：
- **LLM 调用失败直接报错**（C2），不静默降级、不伪装成"这轮没收益"。过去糊进 C1 的后果是
  run 被静默踢进规则模式，还得靠事后取指纹法医式复原"为什么没用 LLM"。C2 穿透后，那套指纹 /
  降级侦测机制大半可删。
- **LLM 模式在装配阶段一次性裁定**（§1 总控）：配置了就必须能用、调用失败即报；没配置就整条 run
  显式走规则。不存在运行中两种模式偷偷来回切。
- **"LLM 给了合法但无用的答案"不是 C2**：那是内容质量问题（C1 邻域），可回规则阶梯——
  与"调用本身失败"严格区分。

## 4. 与现状的差距（待逐项填）

逐项对照 §1 完整性表 + §3 分类法，标出缺哪几项、哪些 C2 被当成 C1 吞了。
**下列改动方向均已确认；`[ ]` 追踪实现进度，尚未动代码。**

- [x] **职责泄漏 + 无损契约**（已实现，commit）：grad 不再判硬停——`hard_stop_reason` 上移总控
      （`loop.hard_stop_reason(state)`，每轮顶部检查预算/轮数/stale）。grad 收窄为只算方向：返回
      `Policy`（迈出的一步）或 `NoMove(reason)`（gradient≈0）。总控读 `NoMove.reason` 直接写
      stop_reason，删掉反扫事件的 `_last_analyze_stop_reason`。停因不再走事件侧信道。
      （用 `Policy | NoMove` 实现 spec §1② 的 `Move | NoMove`，Policy 自身即 Move，省一层包装。）
- [ ] **C2 穿透**：grad / train 的 LLM 失败目前 never-throw 降级 → 改直接报错。
- [ ] **LLM 模式**：目前每轮探测可用性 / 中途降级 → 改装配阶段一次性裁定。
- [ ] **C1 / C2 分离 + 重试策略**：eval 现在把所有失败都翻成同一个 runtime gate →
      按"worker 是否产出结构化裁决"分流：
      - worker 产出裁决（pass/fail）→ 信它，fail = C1，拒候选、继续，**不重试**（eval 确定性）。
      - 候选**超时** = C1（候选太慢 / 挂死），拒、继续，**不重试**（重试再烧一个 timeout 且大概率再超）。
      - worker 起不来 / 非零退出且无 JSON / 输出非法 = 疑似 C2 → **重试一次**；仍失败 → **升级 C2**：
        记基建故障、以 `evaluator_infra_failure` 停 run，但总控仍 finalize 发布当前 best（must-publish 不破）。
      - 前提：worker 要把候选自身异常 catch 成结构化 C1 裁决，只有真·进程级死亡才落 C2 分支。
- [~] **契约合一**（跳过）：澄清后发现并非"实现两遍"——gate 运行期逻辑已单源（quick / full 同走
      worker `gate.run_gate`，只差 mode），quick(被 generate 包成 agent 自检工具) vs full(loop 固定
      门禁) 的内/外切分是**故意保留**的。真正散在三处的是"API 契约知识"（check_self_contained 静态 /
      ENGINE_CONTRACT 文字 / gate 运行期冒烟），是不同目的的不同表示、不强合。API 漂移风险接受，暂跳过。
- [x] **emit 统一**（已实现，commit）：删除两份私有 `_emit`（trainer / grad），调用点直接调唯一的
      共享 `state.loop.emit`（统一补 ts / 默认带 round / 走 add_event 实时 sink；source 默认 "loop"，
      analyze 显式传）。data 跨界键名提成契约一并在 ①②③④ 重构那些键时落。
- [x] **砍候选缓存 + id 简化**（已实现，commit）：删 trainer 去重分支；id 从 `r{round}-{sha1(code)}`
      改成运行内序号 `c{seq}`（seq = generate 数 `candidates/` 目录数，与 LoopState 解耦）；
      state 层 `make_candidate_id(seq)` 保持纯函数。**保留**：同一候选评测幂等（gate 判空）、oracle 参照缓存。
